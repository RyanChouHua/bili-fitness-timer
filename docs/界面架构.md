# 界面架构 — bili-fitness-timer-v2

> 本文是 v2 **界面层的唯一事实来源**,优先级高于早期「重构计划 §5」里的单面板设想。
> 领域逻辑与数据契约仍以 [项目精要.md](项目精要.md) 为准,本文只定义 UI 形态与交互。
>
> 立文背景:旧版「面板一直弄不好」的病根,是**一个悬浮窗同时扛"录入编辑"和"训练运行"两种相反的使用场景**,
> 又要挤在屏幕角落适配三种设备。v2 的界面对策是**按使用阶段拆界面 + 接管 B 站页面**。

---

## 1. 核心原则:按「使用阶段」拆,不按「功能」并列

旧版和 B 的卡 2 产出都是**单面板 + 四 tab(录入/分组/预览/设置)按功能并列**。
v2 改为按真实使用阶段切开:

| 阶段 | 用户在做什么 | 界面需要 | 对应区域 |
|---|---|---|---|
| **准备** | 录入动作、管理分组、调设置、导入导出 | 大空间、键盘、信息看得全,**不看视频** | 工作台抽屉 |
| **训练** | 盯视频做动作、看组次、点完成本组、看倒计时 | 极简、大按钮、余光可读,**几乎不录入** | 训练舱覆盖层 |

这两种场景对界面的要求相反(信息密集 vs 极简),硬塞进一个面板就是旧版反复出问题的根源。
所以第一刀沿「准备 / 训练」这条缝切开。

---

## 2. 三个区域(界面骨架)

v2 界面 = **2 个主形态 + 1 个常驻条**,互斥切换,不再堆 tab。

```
┌─────────────────────────────────────────────┐
│  B 站视频页(被接管)                          │
│  ┌───────────────────────┐                   │
│  │      视频框(保留)      │   ← 训练舱覆盖层  │
│  └───────────────────────┘      遮住此处以外  │
│  ┌───────────────────────┐      的非视频区    │
│  │      评论区(保留)      │                   │
│  └───────────────────────┘                   │
└─────────────────────────────────────────────┘
   [常驻控制条] ← 始终在,负责切换与开关
   [工作台抽屉] ← 从侧边滑出,与训练舱互斥
```

### 区域 A —「训练舱」(训练阶段,覆盖层)

- **形态**:覆盖层(overlay),**全遮罩**——遮住 B 站播放器下方的点赞/投币/收藏、UP 主信息、推荐列表、侧栏,**只留视频框和评论区可见**。
- **z-index** 高于 B 站内容但低于工作台抽屉。
- **内容**(训练必需,极简):
  - 当前动作大标题 + 组次进度(如「深蹲 · 第 2/4 组」)
  - **超大主按钮**:随状态显示 `开始训练` / `完成本组` / `继续` / `休息 12s`(倒计时不可点)
  - 次要控制行:`暂停` `跳过休息`
  - 动作队列迷你预览:当前高亮,锁定时训练中不可切;解锁可点切换
  - 安全区:`重置训练`(训练中点击需二次确认)
  - **休息倒计时视频叠加(竞品对照后新增)**:`rest` 状态下,除面板内的 `休息 Ns` 外,在**视频画面上叠加一个超大倒计时数字**(半透明、不拦截视频原生交互 `pointer-events: none`),让用户做完一组抬头时余光即可读到剩余秒数。这直接服务「余光可读」的核心目标;训练舱本就在视频层,叠加成本低。退出 `rest` 时移除。见 [项目精要.md](项目精要.md) §3「竞品对照后新增候选」。
- **不含**:录入框、分组管理、设置、导入导出。这些全在工作台。
- **设计要求**:大字、大按钮、余光可读;触控目标移动端 ≥ 44px。

### 区域 B —「计划工作台」(准备阶段,侧边抽屉)

- **形态**:从侧边滑出的抽屉(drawer),覆盖在页面上,**不与视频抢原位空间**;打开时可加半透明遮罩。
- **与训练舱互斥**:同一时刻只显示一个。打开工作台 → 训练舱隐藏;开始训练 → 工作台关闭、训练舱显示。
- **内容**(信息密集,可滚动):
  - 录入区:多行 textarea(大)+ 实时解析错误提示
  - 插入当前时间:`插入开始` / `插入结束`(从视频 currentTime 取)
  - 分组管理:列表 + `新建` `复制` `切换` `改名` `删除` + 分页(旧版 pageSize=4)
  - 计划信息:标题 / 作者 / 备注
  - 数据:`导入` `导出` `在线导入`
  - 设置:休息提示音时长(1/2/3/5s)、锁定预览开关
- **设计要求**:给足宽度与滚动空间,不迁就窄条;内部可用轻量分区(折叠或锚点),但**不再用 tab 承载主流程**。

### 区域 C —「常驻控制条」(极小,始终在)

- **形态**:贴边小条(如右下或顶部一角),始终可见,体积极小。
- **内容**:
  - 品牌标 + 当前分组名(如「BF · 周一动作」)
  - `工作台` 开关(滑出/收起抽屉)
  - 主形态切换:准备 ⇄ 训练(等价于"进入训练舱 / 回到准备")
- **职责**:唯一的全局入口。用户从这里在两个形态间切换。

## 3. 交互流程(状态与形态映射)

界面形态由「用户意图」驱动,与训练 `Mode`(idle/exercise/rest/paused/complete)解耦但联动:

| 用户操作 | 形态变化 |
|---|---|
| 首次注入 | 常驻控制条出现;默认显示工作台(若无计划)或训练舱(若已有计划) |
| 点常驻条「工作台」 | 滑出工作台抽屉,隐藏训练舱 |
| 工作台内点「开始训练」 | 关闭工作台,显示训练舱,`Mode: idle→exercise` |
| 训练舱「完成本组」 | 停留训练舱,`Mode→rest`,倒计时 |
| 训练舱「重置」 | 训练中需二次确认;确认后 `Mode→idle`,停留训练舱 |
| 训练中想改计划 | 点常驻条「工作台」→ 滑出抽屉(训练舱隐藏,训练状态保留在内存) |

**互斥规则**:训练舱与工作台不同时显示。训练状态(runtime)在切到工作台时**保留不清**,切回训练舱恢复。

## 4. 接管 B 站页面的技术策略

- **保留元素**:视频框(`<video>` 容器)、评论区。这两块**不被遮罩覆盖**。
- **遮罩范围**:除上述两块外的非视频区(推荐列表、侧栏、播放器下方互动栏、UP 主卡片)由训练舱覆盖层遮住。
- **实现要点**(留给 platform + ui 层,Wave 2/4 接线):
  - 定位视频框与评论区容器的选择器,集中封装在 `platform/bilibili.ts`,B 站改版只改一处。
  - 覆盖层用 `position: fixed` 铺满视口,再"挖空"视频与评论区位置(或把视频/评论提升到覆盖层之上)。
  - 覆盖层 z-index 高于 B 站内容;工作台抽屉 z-index 再高一层。
  - **不修改 B 站 DOM 结构**,只叠加自己的层,避免破坏原生播放器功能(全屏、清晰度、弹幕等)。
- **风险**:B 站 SPA 改版 / 视频容器结构变化。对策:选择器集中封装,提供 fallback(找不到视频框时降级为不遮罩、仅显示常驻条 + 工作台)。

## 5. 响应式(只保留必要断点)

沿用「统一思路 + 极少断点」,但按三区域重新定义:

### 桌面(宽屏)
- 训练舱覆盖层:视频居中/居左,训练控制在视频侧边或下方的遮罩区。
- 工作台抽屉:从右侧滑出,宽度 `min(420px, 90vw)`。
- 常驻条:右下角或顶部一角。

### 移动端 / 平板(`@media max-width: 640px`,含安卓平板 / Via)
- **训练舱贴底部,视频在上**:视频框保持在页面上部,训练控制(大按钮 + 组次)贴在**底部**全宽,拇指可达。
- 工作台抽屉:改为**全屏或底部大抽屉**(移动端侧边抽屉太窄)。
- 常驻条:可并入底部训练条。
- 高度用 `dvh` 防止移动端地址栏遮挡;底部留 `env(safe-area-inset-bottom)`。

> 断点上限:**一个** `@media (max-width: 640px)`。**砍掉** `@container` query、`641-1180` 平板专用段、手动 clamp 尺寸(这些是旧版打架的根源)。

## 6. 明确去掉的东西(相对旧版 / B 的卡 2)

| 去掉 | 原因 |
|---|---|
| 面板**拖动** | 训练舱是覆盖层、工作台是抽屉,位置固定,不需要 |
| 面板**手动缩放** | 同上;旧版此处最容易坏 |
| **收起/展开** 折叠面板 | 用"常驻条 + 互斥形态切换"替代,语义更清晰 |
| **四 tab 并列**(录入/分组/预览/设置) | 拆进训练舱(预览)与工作台(录入/分组/设置),不再用 tab 承载主流程 |
| `panelPosition` / `panelSize` 持久化 | 无拖动缩放,不再需要 |

`Preferences` 里的 `panelPosition` / `panelSize` 字段**保留读取兼容**(旧数据不报错),但 v2 不再写入、不再使用。

---

## 7. 从 B 的卡 2 组件到三区域的映射(给重做的 Codex)

B 已做出的组件大部分可复用,只是**重新编排容器**,不必推倒重写:

| B 现有组件 | 去向 |
|---|---|
| `StatusHeader` `Metrics` `PrimaryAction` `SecondaryControlRow` `SafetyActionZone` | → **训练舱** |
| `PreviewList` | → **训练舱**(改为迷你队列)|
| `EntrySection` | → **工作台**(录入区)|
| `GroupList` + `Pagination` | → **工作台**(分组管理)|
| `SettingsPanel` | → **工作台**(设置)|
| `TabBar` | **废弃**主流程用途;工作台内部若需分区可用轻量折叠 |
| 计划信息表单(标题/作者/备注) | → **工作台**(B 尚未做,需补)|
| 导入/导出/在线导入按钮 | → **工作台**(B 尚未做,需补)|
| 常驻控制条 | **新增**(B 尚未做)|

**保留 B 的好东西**:props 驱动、设计令牌、单列思路、无横向溢出、44px 触控。这些延续到三区域里。

## 8. 预览验收(卡 2 重做后的新验收清单)

`preview/` 需能脱离 B 站展示三区域。状态切换器 + 宽度切换器保留,并新增**形态切换**:

- [ ] 训练舱:idle / exercise / rest / paused / complete 五态,大按钮随态变化,当前动作高亮
- [ ] 训练舱:重置二次确认态
- [ ] 训练舱 rest 态:视频上叠加超大倒计时数字(半透明、不拦截交互)—— 竞品对照后新增,可选带上,不阻塞预览门
- [ ] 工作台抽屉:录入(正常 / 错误提示)、分组列表 + 分页、计划信息表单、导入导出区、设置
- [ ] 常驻控制条:品牌 + 分组名 + 工作台开关 + 形态切换
- [ ] 形态互斥:打开工作台隐藏训练舱,反之亦然
- [ ] 桌面:训练舱遮罩 + 视频/评论留白位、工作台右侧抽屉
- [ ] 移动端(≤640):训练舱贴底、视频在上、工作台全屏/底部大抽屉,无横向溢出、按钮 ≥44px
- [ ] 用模拟的"视频框占位 + 评论区占位"演示遮罩挖空效果

> 预览门仍由**用户拍板**;三区域全部通过才进入功能接线。
