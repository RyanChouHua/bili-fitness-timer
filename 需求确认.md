# 需求确认

来源文件：`修改要求.md`

## 需求范围

1. 适配平板、PC、手机浏览器界面。
   - 目标：脚本界面在 PC、平板、手机浏览器中都可用，不再要求或尝试设置 PC UA。
   - PC 端：保持悬浮面板、小尺寸按钮、可拖拽位置。
   - 平板端：面板宽度适中，保留足够操作区域，按钮不拥挤。
   - 手机端：面板优先底部贴边显示，数据录入、预览、控制按钮适合触控。
   - 验收标准：
     - PC 浏览器下界面不遮挡主要播放器控制。
     - 平板横屏/竖屏下按钮和文字不溢出。
     - 手机浏览器下录入、保存、开始训练、完成本组、休息倒计时均可正常操作。
     - 不再实现或提示 PC UA / 桌面版网站相关能力。

2. 优化界面布局，数据录入和动作预览可以折叠。
   - 数据录入区：包含动作时间戳文本框、插入开始/结束、示例、导入/导出等工具按钮。
   - 动作预览区：包含解析后的动作列表、时间段、组数、次数和休息时间。
   - 目标：在保留完整功能的前提下，让面板层级更清晰、操作区更紧凑、移动端占用高度更低。
   - 布局要求：
     - 顶部状态区只展示当前训练关键状态：当前动作、当前组数、休息倒计时或完成状态。
     - 训练控制区保持常驻可见，避免折叠录入区或预览区后无法完成训练操作。
     - 数据录入区与动作预览区分组展示，分别提供折叠/展开控制。
     - 工具按钮按用途分组：时间插入、示例/导入/导出/在线导入、保存。
     - 动作预览列表信息密度更高，每个动作以一行或紧凑两行展示：名称、时间段、组数、次数、休息时间。
     - PC 端优先保持小悬浮面板，不遮挡播放器主要控制。
     - 平板端面板宽度和按钮排列适配横屏/竖屏，避免按钮过窄或文字溢出。
     - 手机端优先底部抽屉式布局，控制按钮适合触控，长列表可滚动。
   - 验收标准：
     - 数据录入区可单独折叠/展开。
     - 动作预览区可单独折叠/展开。
     - 折叠状态在刷新后保持。
     - 训练控制区不因折叠而不可用。
     - PC、平板、手机下按钮和文本不发生明显重叠或溢出。
     - 手机端面板展开后仍能滚动访问所有功能。

3. 添加脚本数据保存。
   - 目标：录入的动作时间戳保存到脚本本地数据中，下次访问同一视频仍恢复之前内容。
   - 当前实现基础：
     - 已使用 `localStorage` 按当前 B 站视频保存 `rawInput`。
     - 已保存解析后的 `savedExercises` 动作时间戳列表。
   - 需要确认/完善：
     - 增加明确的“保存”按钮或保存状态提示，避免用户不确定是否已保存。
     - 保持自动保存输入内容。
     - 导入/导出 JSON 应包含原始输入和解析后的动作列表。
   - 验收标准：
     - 输入动作时间戳后刷新页面仍能恢复。
     - 同一视频恢复该视频对应的时间戳。
     - 不同视频之间数据互不覆盖。
     - 用户可看到保存状态或手动触发保存。

4. 在 GitHub 仓库中维护在线时间戳库，并按视频 BV 号匹配导入。
   - 目标：仓库增加一个专门目录存放时间戳文件，脚本可根据当前 B 站视频 BV 号在线查找并导入对应时间戳。
   - 建议目录：
     - `timestamps/`
   - 建议文件命名：
     - `timestamps/BVxxxxxxxxxx.json`
     - 文件名中的 BV 号必须与当前视频 URL 中的 BV 号完全一致。
   - 建议 JSON 格式：

```json
{
  "bvid": "BVxxxxxxxxxx",
  "title": "可选：视频标题或训练计划名称",
  "rawInput": "俯卧撑 00:12-00:28 3x8-12 rest45\n深蹲 01:05-01:22 4x10 rest60",
  "exercises": [
    {
      "name": "俯卧撑",
      "start": 12,
      "end": 28,
      "sets": 3,
      "minReps": 8,
      "maxReps": 12,
      "restSeconds": 45
    }
  ]
}
```

   - 在线地址规则：
     - `https://raw.githubusercontent.com/RyanChouHua/bili-fitness-timer/main/timestamps/<BV号>.json`
   - 脚本行为：
     - 进入视频页后提取当前 BV 号。
     - 用户点击“在线导入”时，请求对应 GitHub raw JSON。
     - 如果找到文件，则导入 `rawInput`；若没有 `rawInput`，则从 `exercises` 生成录入文本。
     - 导入成功后写入本地保存，下次访问同一视频无需再次在线导入。
     - 如果文件不存在，提示“未找到该视频的在线时间戳”。
     - 如果网络失败或 JSON 格式错误，提示明确错误，不覆盖本地已有数据。
   - 验收标准：
     - 当前视频 BV 号为 `BVxxx` 时，只请求 `timestamps/BVxxx.json`。
     - 成功导入后，动作列表和录入文本同步更新。
     - 刷新页面后仍保留导入结果。
     - GitHub 上不存在对应 JSON 时，本地数据不被清空。

5. 修正 Bilibili 视频页识别和域名匹配。
   - 背景：当前 `@match` 只写 `https://www.bilibili.com/video/*`，不能准确覆盖 Bilibili 的所有视频网页。
   - 目标：脚本能在常见 Bilibili 视频播放页启用，在非视频页不注入面板。
   - 需要覆盖的 URL 类型：
     - `https://www.bilibili.com/video/*`
     - `https://m.bilibili.com/video/*`
     - 带查询参数的播放页，例如 `?p=2`、播放列表参数等。
     - 其他路径中包含 BV 号且实际存在播放器的页面，例如部分列表/合集播放页。
   - 实现要求：
     - `@match` 可以放宽到 Bilibili 相关域名。
     - 运行时必须从 `location.href` 提取 BV 号。
     - 运行时必须确认页面存在 `video` 元素后再注入面板。
     - 无 BV 号或无播放器时不启用训练面板。
   - 验收标准：
     - 常规 PC 视频页可启用脚本。
     - 手机域名视频页可启用脚本。
     - 带 `p` 参数或播放列表参数的视频页可启用脚本。
     - 非视频页不显示训练面板。

## 非目标

- 不改为 Bilibili-Evolved 插件，继续保持独立 Tampermonkey userscript。
- 不依赖自建后端服务；在线时间戳仅使用 GitHub raw 静态文件。
- 不做用户级云同步、账号同步或多人协作编辑。
- 不实现 PC UA 修改、请求头伪装或引导开启桌面版网站。

## 后续实现建议

1. 增加 UI 折叠状态字段到全局偏好，例如：
   - `inputCollapsed`
   - `previewCollapsed`
2. 将录入区、动作预览区、训练控制区拆成独立渲染块，并按功能分组布局。
3. 增加保存状态文案，例如：
   - `已自动保存`
   - `已手动保存`
   - `当前输入有错误，已保留上一次有效动作列表`
4. 增加响应式断点，例如：
   - PC：`> 1024px`
   - 平板：`721px - 1024px`
   - 手机：`<= 720px`
5. 增加在线导入实现：
   - 添加 `timestamps/` 目录和示例 JSON。
   - 添加“在线导入”按钮。
   - 使用当前 BV 号拼接 GitHub raw URL。
   - 在线导入成功后调用现有本地保存逻辑。
6. 修正 userscript 匹配策略：
   - 放宽 `@match`。
   - 增加运行时 BV 号检测。
   - 增加 video 元素检测。
